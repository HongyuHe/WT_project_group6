Knupp states that the GET method is the most common HTTP method. He does not discuss that the HTTP GET method is assumed to be idempotent. Explain
  1. what idempotent means in this context
  2. how idempotent requests simplify the implementation of a caching HTTP proxy,
  3. how so-called "visitor counters" often break this assumption, and
  4. how such counters might be negatively affected by proxies.
  1. An idempotent HTTP method can be performed multiple times with the same result being produced. In this context that means that the GET method can be called multiple times, with the same result.
  2. Idempotent requests, due to their repeating nature, allow for efficient caching. As idempotent methods will send out the same data, the proxy can cache these requests and get "hits" often.
  3. Visitor counters are a display of how many users are visiting a site. They can be implemented by tracking the IP address of a request or by tracking cookies for instance. Because the state of the web server is changing whilst tracking users, it is breaking the assumption of idempotency, which would have the server's state remain the same.
  4. Proxies could negatively affect visitor counters as they obstruct tracking. Users are more hidden, either in a proxy, or in a reverse proxy system. This way it is harder to tell how many unique users there are, which is the core of a visitor counter
Q2) HTTP is a so-called stateless protocol. Explain what this means. Name at least one advantage and disadvantage of the fact that HTTP is stateless.
HTTP is a stateless protocol meaning that each request must carry all of its information each time it is sent. The server is not tracking each user, and as such cannot associate different requests with a single user. One negative of this statelessness is the fact that a user must send a lot of repeating information in its requests, such as what browser they are using, all of which is increasing the size of a packet. However a positive aspect of this statelessness comes from the simplicity that arrives with it. A stateless webserver is much simpler to build, and is a lot less complex computationally, than one that would track states. Of course using HTTP, we cannot build a stateful web server except by using workarounds such as cookies.

6. Explain why some prefer the the HTML of the first paragraph over the second, and give an example where this could be important. Also, identify another pair of HTML constructs for which the same rationale holds.

I think it is because <b> only refers to style, which means we use <b> to make the text look bold purely for stylistic reasons. Meanwhile we use <strong> if we wanted that bit of text to look bold and have emphasis. That means that <strong> indicates an important piece of information. In other words, <strong> is a semantic tag (carrieis meaning), whilst <b> is not semantic.

Example: In terms of you looking at the output on a page, there is no visual difference. They look exactly the same. The difference comes with text-to-speech programs, often used by blind people. When blind people look at a website, all the text is spoken to them. If you use <b>, the voice program will not do anything differently when reading the text. But if you use <strong>, the voice program might speak the text differently—with emphasis, or in a different tone of voice. This is done to convey to the blind person that the text is important in some way.

5. Summarize the key multilingual issues that W3C (Links to an external site.) Links to an external site. is addressing within the "Internationalization" activity.

The multilingual issues that the web is facing and that the W3C is trying to resolve are plentiful. The root of the issues stem from the fact that languages around the world are all very different. Just because the web started out in English does not mean that it should be limited to that one language. After all, how could the web truly be "World Wide" if only one language was involved? Because this is the case many groups, including the W3C aim at involving other languages in the web. This is what internationalization refers to. The issues are plenty but we've listed a few here:

  • Character Sets
    Whilst the English alphabet consists of 26 letters, this is certainly not the case for other languages. Some alphabetical systems, such as Korean, have a different amount of letters in their alphabet, or some, like Chinese, don't use an alphabetical system, instead consisting of thousands of characters. As such the W3C tasks itself with correctly displaying these foreign characters on the web. This is difficult to do, especially with encorporating fonts and the rest.
  • Reading order
    In English we read left to right, but in certain languages that is not the case. For example, Japanese is read right to left, or even sometimes top to bottom. This creates problems with displaying this text on websites. By simply using standards meant for the English language this would be difficult, this is why the W3C is interested in standartising bidirectional text.
  • Time Zones
    Time zones are a large issue in internationalization as not all countries are in the same time zone. Because of this, when time is displayed on a web app or web page, there is a possibility that it is the incorrect time for a user in a different time zone. There are many ways to address this issue including using floating time. Floating time refers to time that is not tied to a specific time zone. By using this time format we can altogether avoid the issue of time zones. However almost nobody uses a floating time system. As such a better solution could be location service tracking which can convert between time zones in order to always display the correct time. This is what is used most commonly nowadays.

7. Look up information about the new protocol for IP addresses (IPv6, IP version 6). Describe the main features of this protocol and indicate what the reasons are to move from version 4 to version 6.

Main features of IPv6:

  • Larger Address Space
    In contrast to IPv4, IPv6 uses 4 times more bits to address a device on the Internet. This much of extra bits can provide approximately 3.4×1038 different combinations of addresses. This address can accomodate the aggressive requirement of address allotment for almost everything in this world. According to an estimate, 1564 addresses can be allocated to every square meter of this earth.
  • Simplified Header
    IPv6’s header has been simplified by moving all unnecessary information and options (which are present in IPv4 header) to the end of the IPv6 header. IPv6 header is only twice as big as IPv4 provided the fact that IPv6 address is four times longer.
  • Auto-configuration
    IPv6 supports both stateful and stateless auto configuration mode of its host devices. This way, the absence of a DHCP server does not put a halt on inter segment communication.
  • Faster Forwarding/Routing
    Simplified header puts all unnecessary information at the end of the header. The information contained in the first part of the header is adequate for a Router to take routing decisions, thus making routing decision quickly as it only looks at the important part of the header.
  • IPSec
    Initially it was decided that IPv6 must have IPSec security, making it more secure than IPv4. This feature has now been made optional.
  • No Broadcast
    Though Ethernet/Token Ring are considered as broadcast network because they support Broadcasting, IPv6 does not have any broadcast support any more. It uses multicast to communicate with multiple hosts.
  • Anycast Support
    This is another characteristic of IPv6. IPv6 has introduced Anycast mode of packet routing. In this mode, multiple interfaces over the Internet are assigned the same Anycast IP address. Routers, whilst routing, send the packet to the nearest destination.
  • Mobility
    IPv6 was designed to keep mobility in mind. This feature enables hosts (such as mobile phones) to roam around in different geographical areas and remain connected with the same IP address. The mobility feature of IPv6 takes advantage of auto IP configuration and Extension headers.
  • Enhanced Priority Support
    IPv4 used 6 bits DSCP (Differential Service Code Point) and 2 bits ECN (Explicit Congestion Notification) to provide Quality of Service but it could only be used if the end-to-end devices support it, that is, the source and destination device and underlying network must support it. In IPv6, Traffic class and Flow label are used to tell the underlying routers how to efficiently process the packet and route it.
  • Smooth Transition
    Large IP address scheme in IPv6 enables us to allocate devices with globally unique IP addresses. This mechanism saves IP addresses and NAT is not required. So devices can send/receive data amongst each other, for example, VoIP and/or any streaming media can be used much efficiently. The other fact is that the header is less loaded, so routers can take forwarding decisions and forward them much faster.
  • Extensibility
    One of the major advantages of IPv6 header is that it is extensible, meaning it is possible to add more information in the option part of the packet. IPv4 provides only 40-bytes for options, whereas options in IPv6 can be as much as the size of IPv6 packet itself.

The reasons of moving from version 4 to version 6:

  1. Publicly available IPv4 addresses have been exhausted.
  2. Technological innovation has become the key to business success in the 21st century, so IPv6 is necessary for business growth.
  3. Unlike an IPv4 network in which fragmentation is handled by the router, source devices handle fragmentation in an IPv6 network and they are also responsible for discovering the paths maximum transmission unit. So, by reducing the size of routing tables, IPv6 makes routing more efficient and hierarchical. ISPs can aggregate the prefixes of their customers networks into a single prefix that they can announce to the IPv6 Internet.
  4. Unlike IPv4, IP-level checksum is not included in IPv6, the new Internet Protocol can handle packets more efficiently and deliver better performance.
  5. In IPv6, multicast entirely replaces broadcast. This makes multicast an integrated and necessary function of an IPv6 network. The benefit to this is that multicast saves network bandwidth by allowing bandwidth-intensive packet flows to be sent to multiple destinations simultaneously. While, in IPv4, IPv4, multicast was primarily used to deliver multimedia on IPv4 networks.
